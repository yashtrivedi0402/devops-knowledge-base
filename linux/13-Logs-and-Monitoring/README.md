# 📜 Linux Logs & Monitoring

> **Logs record what happens inside a Linux system, while Monitoring continuously checks the health and performance of the system.**
>
> Logs help identify past events, whereas monitoring helps detect issues in real time. Together, they are essential for troubleshooting, security, and maintaining production servers.

# ❓ What are Logs?

**Logs** are records of events generated by the operating system, applications, and services.

They help track:

* User logins
* Service activity
* Errors
* System events
* Security events

---

# ❓ What is Monitoring?

**Monitoring** is the continuous observation of system resources and services.

It helps monitor:

* CPU Usage
* Memory Usage
* Disk Usage
* Network Traffic
* Running Services
* Server Availability

---

# 🎯 Why Do We Need Logs & Monitoring?

They help to:

* Troubleshoot problems
* Detect failures early
* Improve performance
* Monitor system health
* Audit user activities
* Enhance security

Without logs and monitoring, identifying the root cause of issues becomes difficult.

# 📂 Important Log Files

| Log File            | Purpose                           |
| ------------------- | --------------------------------- |
| `/var/log/syslog`   | General system logs (Ubuntu)      |
| `/var/log/messages` | General system logs (RHEL/CentOS) |
| `/var/log/auth.log` | Authentication logs               |
| `/var/log/kern.log` | Kernel logs                       |
| `/var/log/dmesg`    | Boot messages                     |
| `/var/log/nginx/`   | NGINX logs                        |
| `/var/log/mysql/`   | MySQL logs                        |

# 📖 Viewing Logs

View complete log:

```bash
cat /var/log/syslog
```

View latest entries:

```bash
tail -f /var/log/syslog
```

View service logs:

```bash
journalctl -u nginx
```

View boot logs:

```bash
journalctl -b
```

---

# 📊 System Monitoring Tools

Monitor processes:

```bash
top
```

Interactive monitoring:

```bash
htop
```

Memory usage:

```bash
free -h
```

Disk usage:

```bash
df -h
```

CPU statistics:

```bash
vmstat
```

---

# 🔄 Log Rotation

Linux uses **logrotate** to automatically:

* Rotate old logs
* Compress log files
* Delete outdated logs
* Prevent disks from filling up

Configuration:

```text
/etc/logrotate.conf
```

# 🏭 Production Example

A web application returns **500 Internal Server Error**.

Investigation:

Check service status:

```bash
systemctl status nginx
```

View application logs:

```bash
journalctl -u nginx
```

Check disk usage:

```bash
df -h
```

Review authentication logs:

```bash
tail -f /var/log/auth.log
```

The logs reveal the root cause, allowing the issue to be fixed quickly.

# 🎯 Common Interview Questions

### What are Linux logs?

Records of system and application events used for troubleshooting and auditing.

---

### Where are Linux logs stored?

Most logs are stored in:

```text
/var/log
```

---

### What is `journalctl`?

A command used to view logs collected by **systemd-journald**.

---

### What is logrotate?

A utility that automatically rotates, compresses, and removes old log files.

---

### Which command monitors logs in real time?

```bash
tail -f /var/log/syslog
```
